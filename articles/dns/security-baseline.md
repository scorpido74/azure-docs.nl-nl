# <a name="azure-security-baseline-for-azure-dns"></a>Azure-beveiligings basislijn voor Azure DNS

De Azure-beveiligings basislijn voor Microsoft Azure DNS bevat aanbevelingen waarmee u de beveiligings postuur van uw implementatie kunt verbeteren. De basis lijn voor deze service wordt opgehaald uit de [Azure Security Bench Mark-versie 1,0](../security/benchmarks/overview.md), die aanbevelingen biedt over hoe u uw cloud oplossingen kunt beveiligen in azure met onze richt lijnen voor best practices. Zie [overzicht van Azure Security-basis lijnen](../security/benchmarks/security-baselines-overview.md)voor meer informatie.

*Zie [Azure Security Bench Mark: Logging and monitoring](/azure/security/benchmarks/security-control-logging-monitoring)(Engelstalig) voor meer informatie.*

### <a name="22-configure-central-security-log-management"></a>2,2: Centraal beveiligings logboek beheer configureren

**Hulp**: het activiteiten logboek is een platform logboek in azure dat inzicht biedt in gebeurtenissen op abonnements niveau. De logboeken verzenden naar een Log Analytics-werk ruimte, een Azure Event Hub of een Azure Storage-account voor archiveren. Het activiteiten logboek biedt inzicht in de bewerkingen die zijn uitgevoerd op uw Azure DNS resources op het niveau van het besturings element. Met Azure-activiteiten logboek gegevens kunt u bepalen wat voor schrijf bewerkingen (PUT, POST, DELETE) die zijn uitgevoerd op het niveau van de Control level voor uw DNS-zones, de ' wat, wie en wanneer ' zijn.

Opname logboeken via Azure Monitor voor het verzamelen van beveiligings gegevens die zijn gegenereerd door eindpunt apparaten, netwerk bronnen en andere beveiligings systemen. U kunt ook gegevens in-en inschakelen voor Azure Sentinel of een SIEM van derden.
